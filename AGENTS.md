# Правила экономии токенов (Token Saving Rules)

Для оптимизации использования токенов и ускорения работы соблюдайте следующие правила:

1. **Лаконичность ответов**:
   - Пишите ответы максимально кратко и по делу. Избегайте длинных вступлений, вежливых формулировок и подробных объяснений очевидного кода.
   - Не извиняйтесь за ошибки, а просто исправляйте их.
   - После создания или обновления артефакта (например, плана или отчета) не дублируйте его содержимое в чате. Просто дайте ссылку на файл и укажите ключевые вопросы.

2. **Ограничение просмотра файлов**:
   - При чтении файлов с помощью `view_file` всегда указывайте точные диапазоны строк (`StartLine` и `EndLine`), если файл больше 50–100 строк. Не загружайте весь файл целиком без необходимости.

3. **Локальные и точечные правки**:
   - При редактировании файлов используйте `replace_file_content` или `multi_replace_file_content` для замены минимально возможных блоков кода. Не заменяйте большие секции файлов, если изменения затрагивают лишь несколько строк.

4. **Ограничение вывода команд**:
   - Ограничивайте вывод запускаемых в терминале команд. Например, используйте `git log -n 5`, `head`, `grep` или фильтрацию вывода, чтобы не засорять контекст логами большого объема.

5. **Оптимизация вызовов инструментов**:
   - Не делайте повторяющиеся или избыточные вызовы (например, повторный `list_dir` или `grep_search` с теми же параметрами).
   - Сначала анализируйте имеющуюся информацию из предыдущих шагов.
6. Не сканируйте проект повторно без необходимости.
Используйте результаты предыдущего анализа. Не выполняйте повторный list_dir, find или grep, если структура проекта уже известна.

7. Не перечитывайте неизменённые файлы.
Если файл не изменялся и уже был проанализирован, считайте ранее полученную информацию актуальной.

8. Перед изменениями сначала составьте краткий план.
Если задача затрагивает более трёх файлов — сначала покажите план в 3–5 пунктов, затем приступайте к работе.

9. Не создавайте новые файлы без необходимости.
По возможности изменяйте существующие файлы.

10. После выполнения задачи выводите только:
- какие файлы изменены;
- краткое описание изменений;
- результат проверок.
